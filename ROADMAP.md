# QueryPaw Roadmap

[中文](#中文) | [English](#english)

## 中文

这个 Roadmap 用来帮助用户和贡献者了解项目方向。它不是承诺清单，优先级会随着真实反馈调整。

### 近期

- 补充 README 截图、演示 GIF 和快速上手示例。
- 提升 Windows 分发体验，包括安装包、winget、Scoop 和 Chocolatey。
- 完善 GitHub Issues、Discussions、Release notes 和贡献者文档。
- 收集不同数据库版本的兼容性反馈，尤其是 MariaDB、SQLite、Dameng、KingbaseES、Oracle、SQL Server、PostgreSQL 和 MySQL。
- 打磨查询结果表格、注释维护、连接管理和 SQL 编辑器的细节体验。

### 中期

- 稳定表结构设计、对象详情视图和模型关系图。
- 增强结果编辑、数据导入导出和批量操作能力。
- 优化 SQL 补全、中文备注辅助查询和元数据缓存。
- 梳理可测试的核心服务，减少主窗口和大型 ViewModel 的复杂度。
- 增加更多数据库兼容性验证和问题诊断能力。

### 长期

- 发布更完整的跨平台包。
- 评估 Microsoft Store/MSIX 分发。
- 建立示例数据库和可重复演示场景。
- 提供更完整的插件化或扩展点设计。
- 在保持轻量的前提下，持续提升开发、测试和运维场景的日常效率。

### 适合新贡献者的方向

- 补充某个数据库版本的连接和查询兼容性说明。
- 修正文档、错别字、英文表达或截图说明。
- 为已有功能补充 Issue 复现步骤或测试数据。
- 改进安装说明、Release notes 或包管理器 manifest。
- 优化小而明确的 UI 细节。

## English

This roadmap helps users and contributors understand the project direction. It is not a fixed promise list; priorities may change based on real feedback.

### Near Term

- Add README screenshots, demo GIFs, and quick-start examples.
- Improve Windows distribution through installer, winget, Scoop, and Chocolatey.
- Improve GitHub Issues, Discussions, release notes, and contributor documentation.
- Collect compatibility feedback for different database versions, especially MariaDB, SQLite, Dameng, KingbaseES, Oracle, SQL Server, PostgreSQL, and MySQL.
- Polish result grids, comment maintenance, connection management, and SQL editor details.

### Mid Term

- Stabilize table design, object details, and model diagram features.
- Improve result editing, data import/export, and batch operations.
- Enhance SQL completion, comment-assisted querying, and metadata caching.
- Move testable core behavior into narrower services and reduce main-window/ViewModel complexity.
- Add more compatibility validation and diagnostics for database providers.

### Long Term

- Publish more complete cross-platform packages.
- Evaluate Microsoft Store/MSIX distribution.
- Build sample databases and repeatable demo scenarios.
- Explore a clearer plugin or extension model.
- Keep improving daily developer, tester, and operator workflows while staying lightweight.

### Good First Contribution Areas

- Add compatibility notes for a specific database version.
- Fix documentation, wording, screenshots, or examples.
- Add reproduction steps or sample data for existing issues.
- Improve installation notes, release notes, or package-manager manifests.
- Polish a small and well-scoped UI detail.
